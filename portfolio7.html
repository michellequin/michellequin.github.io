Components Library

Figma components library of Columbia University's Design System

Spring 2020, Columbia University IT Department (CUIT)

As an assistant UX developer for Columbia University's IT department (CUIT) in Spring 2020, my main project was to create a components library in Figma to use for future website/app prototyping. I reproduced the entire Columbia Design System as easy-to-use Figma components, creating 105 components as well as establishing typeface and colors for standardized usage across university mobile and desktop platforms. Many components include responsive animations and/or allow for customization where appropriate.

I also helped update the Columbia Design System, administrative systems and older websites to be more accessible, uniform and streamlined during my time at CUIT. In addition, I designed and implemented a CUIT website for its UX services and portfolio using Drupal 8.

Technology used

  • Figma
  • Adobe Creative Suite
  • Drupal 8

Visit Components Library


To create components, I began by labeling and alphabetically organizing frames according to the Columbia Design System elements (ex. Autocomplete, Cards, Hero, UI Feedback, etc.). I then made the building blocks of each element. For instance, for Autocomplete, I made a search icon, search bar, search bar text, etc. and formatted dimensions, typeface and colors accordingly. I then grouped all of these parts into a Figma component, which now appears under Assets as a 'local component'.



Where needed, I added animations to components to make them automatically interactive for the sake of future prototyping use. For example, clicking on the 'info' button for the Tooltip component causes a text bubble to pop up. Clicking the 'menu' button for the Mobile Search Bar component causes a dropdown menu to appear, and the 'menu' button is replaced with an 'X' button that closes the dropdown upon clicking. Components can also be customized where needed in terms of text, color or state due to the way I broke down component building blocks, enabled instance swaps and made child components.



Many components have variations, such as Buttons which have different forms depending on if they're primary or secondary as well as if they're in hover state or being clicked. For such components, I made sure to make clearly designated subheadings so that the library is easy to search and navigate. I also gave each component a short description/alt text for ease of reference. To use a component, simply find it in the library and drag it to your prototype layer.
